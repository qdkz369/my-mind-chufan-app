# 解决服务器启动问题

## ✅ 已完成的修复

1. **终止占用端口的进程** - 进程 18604 已终止
2. **检查锁文件** - 锁文件不存在（正常）

## 🔧 现在请手动操作

### 方法一：删除 `.next` 目录（推荐）

在项目根目录下，删除 `.next` 文件夹：

**PowerShell:**
```powershell
Remove-Item -Recurse -Force .next
```

**或者手动删除：**
- 在文件资源管理器中找到项目文件夹
- 删除 `.next` 文件夹

### 方法二：直接重启服务器

如果 `.next` 文件夹不存在或已删除，直接运行：

```bash
npm run dev
```

## 📋 完整步骤

1. **删除 `.next` 目录**（如果存在）
   ```powershell
   Remove-Item -Recurse -Force .next
   ```

2. **重新启动服务器**
   ```bash
   npm run dev
   ```

3. **确认服务器启动成功**
   - 应该看到：`Local: http://localhost:3000`
   - 不应该再看到锁文件错误

## ⚠️ 如果仍然有问题

### 检查是否有其他 Node 进程

如果仍然提示端口被占用，可以：

1. **查找占用 3000 端口的进程**
   ```powershell
   netstat -ano | findstr :3000
   ```

2. **终止该进程**（替换 PID）
   ```powershell
   taskkill /F /PID <PID>
   ```

3. **或者使用其他端口**
   - Next.js 会自动使用 3001、3002 等可用端口

---

**删除 `.next` 文件夹后，重新运行 `npm run dev` 应该就能正常启动了！** 🚀
